# Semantic Model Overview

## Semantic Model

`sem_stm_transit_operations`

## Purpose

Provides business-ready transit KPIs and operational reporting structure.

## Example KPIs

- Active vehicles
- Active routes
- Data freshness
- Stale vehicle count
- Service disruption count
- Records received
- Last successful refresh
- Average latency
- Maximum latency

## Dashboard

The semantic model supports the STM Transit Intelligence Dashboard.
