# 06 Data Quality Checks

## Purpose

Documents the data quality validation logic for the transit data pipeline.

## Checks

- Missing vehicle_id
- Missing route_id
- Missing trip_id
- Invalid latitude or longitude
- Duplicate records
- Stale records
- Ingestion latency
- Out-of-area coordinates

## Example KPIs

- Data quality pass rate
- Stale vehicle count
- Missing coordinate count
- Latest ingestion time
